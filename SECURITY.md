# Security Policy

## Project status

Nexo is currently in planning and pre-alpha. There are no supported production releases yet. Security reports about documentation, future code, dependencies, authentication, authorization, synchronization, or data handling are still welcome.

## Reporting a vulnerability

Please do not disclose a suspected vulnerability in a public issue, discussion, pull request, or social media post.

Use GitHub's private vulnerability reporting feature for this repository when it is available. If private reporting is not available, open a minimal public issue asking for a private contact channel without including technical details, proof-of-concept material, credentials, or personal data.

Include, when possible:

- A concise description of the issue
- The affected component or planned capability
- Reproduction conditions
- Potential impact
- Suggested mitigation
- Whether the issue has been disclosed elsewhere

## Sensitive information

Never submit:

- Passwords, access tokens, API keys, or signing credentials
- Real household, health, financial, location, or identity information
- Private logs or database exports
- Exploitable proof-of-concept material in a public channel

## Response expectations

This is an early personal project without a formal security response team or guaranteed service-level agreement. Reports will be acknowledged and assessed as capacity allows. Valid issues will be handled before public technical disclosure whenever practical.

## Security goals

Nexo intends to treat the following as core requirements:

- Private-by-default data
- Server-side authorization
- Least-privilege household roles
- Secure credential storage
- Encrypted transport
- Session revocation
- Data export and deletion
- Dependency review
- Auditability for sensitive household changes

Thank you for helping keep future Nexo users and their data safe.
